# 智造课题库

用于展示单片机毕业设计项目的静态网站。唯一数据源是 `data/项目链接清单.xlsx`，替换并上传该文件后，GitHub Actions 会自动校验、转换和发布 GitHub Pages。

## 项目目录

```text
毕业设计资料库_/
├─ data/
│  └─ 项目链接清单.xlsx      # 唯一上传数据源，项目必需
├─ site/                     # GitHub Pages静态网站
│  ├─ index.html
│  ├─ app.js
│  ├─ styles.css
│  ├─ assets/
│  │  ├─ customer-wechat.png
│  │  └─ generated/projects/ # Excel图片自动提取，禁止手工修改
│  └─ data/
│     └─ projects.json       # Excel自动生成，禁止手工修改
├─ scripts/                  # Excel解析和本地预览脚本
├─ test/                     # Excel校验和网页搜索测试
├─ .github/workflows/        # GitHub自动解析与部署流程
├─ package.json              # 本地预览命令
├─ requirements.txt          # Excel解析依赖
├─ README.md                 # 使用说明
└─ 1/                        # 备份、旧版和临时文件，不参与项目运行
```

`1/` 已加入 `.gitignore`，不会上传到 GitHub。

## Excel 字段

工作表名称保持为 `项目链接清单`，第一行是表头，每行对应一个项目。

| 字段 | 是否必填 | 填写说明 |
|---|---:|---|
| 序号 | 否 | 便于人工查看 |
| 项目系列 | 否 | 如 Y系列、S系列STM32单片机 |
| 项目编号 | 是 | 同一项目系列内不能重复 |
| 项目名称 | 是 | 客户看到的项目标题 |
| 单片机分类 | 是 | 如 STM32、51单片机、ESP32 |
| 单片机型号 | 否 | 如 STM32F103C8T6、STC89C52 |
| 项目用途 | 否 | 多项用顿号或逗号分隔 |
| 使用模块 | 否 | 多项用顿号或逗号分隔 |
| 仿真图片 | 否 | 使用WPS“嵌入单元格”图片，每个单元格一张 |
| 实物图片 | 否 | 使用WPS“嵌入单元格”图片，每个单元格一张 |
| 项目简介 | 否 | 简短功能介绍 |
| 仿真+仿真代码 | 否 | 填金额、面议、咨询；空白或“无”表示没有该方案 |
| 原理图+PCB设计 | 否 | 填金额、面议、咨询；空白或“无”表示没有该方案 |
| 硬件实物+配套硬件代码 | 否 | 填金额、面议、咨询；空白或“无”表示没有该方案 |
| 论文 | 否 | 填金额、面议、咨询；空白或“无”表示没有该方案 |
| 是否展示 | 否 | 填“是”或“否”，默认为“是” |
| 排序 | 否 | 整数，越小越靠前 |

图片尺寸不要求一致。网站会使用固定画框完整显示，不裁剪、不拉伸；点击图片可查看原图。没有图片的项目不会显示空白图片区。

网站卡片会显示最低价格，例如 `¥100 起`；价格组合默认展开，可自由勾选多个方案、填写备注并实时计算总价。确认需求后可一键复制项目名称、编号、需求组合、报价和备注，并扫码添加客服微信。空白或“无”的价格方案不会显示，也不参与计算；四种方案都不可用时不显示报价控件。

## 隐私说明

- 公开 Excel 不允许包含`下载链接`、`资料介绍链接`和`资料主要内容`等内部字段。
- 这些字段也不会进入 `projects.json`，网页不会展示任何下载地址或内部资料。
- 备份和旧版文件统一放在 `1/`，不要放在项目根目录。

## 日常更新方式

1. 打开 `data/项目链接清单.xlsx`。
2. 新增项目、嵌入图片、修改价格或将“是否展示”改为“否”，然后保存。
3. 在 GitHub 仓库中进入 `data` 文件夹，点击 `Add file → Upload files`。
4. 上传并替换同名文件 `项目链接清单.xlsx`。
5. 点击 `Commit changes`，等待约 1～3 分钟。

不需要手工修改 JSON，不需要 WPS、飞书、API 或 GitHub Secrets。

## 图形上传工具

Windows 下可以双击 `上传工具/启动上传工具.bat`。选择 Excel 后先点击“检测数据表”；有问题会在日志区域显示具体原因，检测通过后才能点击“上传到 GitHub”。工具只提交 `data/项目链接清单.xlsx`，不会提交其他文件。

## 自动校验

发布前会检查：

- 必填表头和必填单元格
- 同一项目系列内的编号重复
- 排序字段是否为整数
- 价格是否为非负金额、面议、咨询、无或空白
- WPS单元格图片是否完整，格式是否为JPG或PNG，单张是否超过20MB
- 是否包含禁止公开的链接或资料内容列
- 是否至少存在一个可展示项目

校验失败时不会发布新版本，现有网站继续正常运行；具体错误行会显示在 Actions 日志中。

## 本地预览

直接打开 `site/index.html` 即可查看现有页面。需要本地服务器时：

```powershell
npm start
```

然后打开 `http://127.0.0.1:4173`。这个地址只在命令运行期间有效。

修改 Excel 后，如需本地重新生成网页数据：

```powershell
python -m pip install -r requirements.txt
python scripts/excel_to_json.py
```

## GitHub Pages

首次上传仓库后，在 `Settings → Pages → Source` 中选择 **GitHub Actions**。以后只要替换 `data/项目链接清单.xlsx`，网站就会自动更新。
